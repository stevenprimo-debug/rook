# Creative Director — Frameworks Index (callable methodologies)

Each framework is a runnable operation the agent invokes. Originator credit lives in
[`frameworks_attribution.md`](frameworks_attribution.md) — never invoked in output.

---

## Provocation-Pole methodologies

### `art_science_intersection(project)`

**Input:** project brief + current work-in-progress.
**Output:** verdict on whether the work is BOTH technically rigorous AND aesthetically
resolved. Flags either-side failure.
**Rule:** "Both, or neither — there is no third option that ships."

### `manifest_destiny_demo(project)`

**Input:** the standard-scope version of the artifact.
**Output:** the most ambitious version that is *demonstrably possible* with current
constraints. Names the gap between standard and ambitious.
**Rule:** "If it is not manifestly important and nearly impossible, the brief is too
small."

### `belief_specificity_check(brief)`

**Input:** brief + position statement.
**Output:** confidence score on whether the position is specific enough to be
polarizing. Returns rewrite candidates that raise the specificity.
**Rule:** "If the brand wouldn't lose anyone with this position, the position isn't
real."

## Restraint-Pole methodologies

### `source_listening(brief)`

**Input:** brief + raw context dump.
**Output:** one-paragraph statement of what the work is actually about, stripped of
decoration, requirements lists, and stakeholder noise.
**Rule:** "The first answer is rarely the source. Listen past it."

### `reduce_to_essence(work)`

**Input:** current artifact (deck, page, copy, design).
**Output:** the single load-bearing element. Flags every other element as a candidate
for removal.
**Rule:** "Subtract until removing one more thing would break it."

### `four_phases_audit(work)`

**Input:** work-in-progress.
**Output:** identifies which of four creative phases the work is in — `seed` /
`experiment` / `craft` / `release` — and prescribes what the phase needs (and forbids
what it doesn't).
**Rule:** "Phase mismatch is the most common failure. Craft-phase polish on
seed-phase ideas kills the seed."

## Coherence-Pole methodologies

### `belief_extraction(work)`

**Input:** any work artifact.
**Output:** the load-bearing belief the artifact is built on, named in one sentence.
Flags absence (work without a belief is decoration).
**Rule:** "If you can't name the belief in one sentence, the work doesn't have one."

### `pace_layering(decision)`

**Input:** a creative decision.
**Output:** which layer the decision belongs to — `fashion` (3 months) / `commerce` (3
years) / `infrastructure` (30 years) / `governance` / `culture` / `nature`. Catches
fashion-layer decisions being made on infrastructure-layer artifacts.
**Rule:** "Match the pace of the decision to the pace of the artifact."

### `access_to_tools_audit(brief)`

**Input:** brief + maker capacity.
**Output:** does the maker have what they need to execute. Identifies missing tools,
references, or context. Returns a request-for-tools artifact.
**Rule:** "A brief without tooling is a wish."

## Cross-pole (synthesis) methodologies

### `creative_brief(project)`

**Input:** project context.
**Output:** the brief that names what the work is for, in four sections:
- **BELIEVE** — the load-bearing position
- **REJECT** — what the work refuses
- **FEEL** — the emotional contract with the audience
- **SUSTAIN** — the throughline that keeps every element consistent

### `taste_calibration(reference, work)`

**Input:** reference corpus + current work.
**Output:** confidence score that the work hits the calibration bar set by the
reference. Flags drift.
**Rule:** "Your taste is the average of your last 100 references. Choose them."
