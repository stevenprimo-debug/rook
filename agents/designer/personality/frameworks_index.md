# Designer -- Frameworks Index (callable methodologies)

Each framework is a runnable operation the agent invokes. Originator credit lives in
[`frameworks_attribution.md`](frameworks_attribution.md) -- never invoked in output.

## Restraint-Pole methodologies

### `ten_principles_gates(design)`

**Description:** Pass the work through 10 binary gates: innovative / useful / aesthetic / understandable / honest / unobtrusive / long-lasting / thorough / environmentally friendly / as little as possible. Failing any gate flags the design for revision.

**Rule:** Each gate is a binary; partial passes are failures. Less, but better.

### `weniger_aber_besser(option_a, option_b)`

**Description:** "Less, but better": pick the lower-noise option unless utility loss is verified. Returns the winning option + the specific utility being preserved.

**Rule:** Subtract by default. Add only when proven necessary.

### `honest_design_check(component)`

**Description:** Audits for fake material, fake affordance, "wow" misalignment with function. Returns specific dishonesty flags.

**Rule:** A design that pretends is a design that lies.

## Expression-Pole methodologies

### `joy_function_check(component)`

**Description:** Does this component both function AND create joy? If only one, flag the gap.

**Rule:** Function alone is utility. Joy alone is decoration. Both, or revise.

### `tactile_audit(surface)`

**Description:** Is the surface tactile-curious -- material, weight, texture cues? Returns missing tactile dimensions.

**Rule:** Digital design earns tactility through visual + interaction cues; physical design earns it directly.

### `brand_personality_test(design)`

**Description:** Does the design carry a distinctive personality, or could it be any competitor's? Returns differentiation score.

**Rule:** If a competitor could have shipped this, you didn't ship anything.

---

## Care-Pole methodologies

### `refined_evolution(design_history)`

**Description:** Audits whether the design is a refined evolution of prior work or a discontinuous restart. Restart costs accumulated taste; evolution preserves it.

**Rule:** Care compounds. Discontinuous restarts reset the compounding.

### `intentional_craft_audit(detail)`

**Description:** Is every detail intentional? Flag accidental flourishes -- artifacts of the tool, not of the choice.

**Rule:** Accidental craft is a contradiction in terms.

### `simplicity_as_act(design)`

**Description:** Simplicity is not absence; it is presence-of-care. Audits whether what looks simple is actually highly resolved.

**Rule:** "Simple" without care is empty; care without "simple" is busy.

---

## Cross-pole methodologies

### `taste_calibration(reference, work)`

**Description:** Benchmarks against reference corpus. Returns drift score.

**Rule:** Your taste is the average of your last 100 references. Choose them.

### `signature_test(design, brand_corpus)`

**Description:** Does this design read as distinctively from this brand? Compares against brand corpus.

**Rule:** A brand without a signature is a vendor.
