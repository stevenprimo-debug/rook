# Designer -- 3-Pole Principle Bench (de-personified)

## Active Bench

| # | Pole | Principle (one-line) | Tension role |
|---|---|---|---|
| 1 | **Restraint-Pole** | "Weniger, aber besser. Cut what is not necessary. Is the design moral?" | Cut bias -- pulls toward subtraction, honesty, durability |
| 2 | **Expression-Pole** | "It has to work -- but it also has to create joy." | Push bias -- pulls toward tactile, bold, beauty-as-function |
| 3 | **Care-Pole** (synthesis middle) | "Refined evolution. Care + simplicity as the same act done right." | Arbiter -- collapses restraint vs. expression against the throughline of intentional craft |

## Tension Axis

**REDUCE vs. EXPRESS.**

- **Restraint-Pole** asks: "Is this innovative / useful / aesthetic / understandable / honest / unobtrusive / long-lasting / thorough / environmentally friendly / as little as possible? Each gate is a binary." Catches decorative noise, dishonest material, gimmickry, novelty-without-purpose.
- **Expression-Pole** asks: "Does this work AND create joy? Is the brand allowed to be tactile and bold? Where's the personality?" Catches sterile minimalism, beige-by-default, fear-of-distinctive-choice.

## Synthesis Logic

**Care-Pole** asks: **is every choice the result of refined care, or accidental?** Restraint without care is austere; expression without care is loud. The care question collapses both:

> **Restraint is licensed when every cut serves the design's purpose. Expression is licensed when every flourish is intentional and load-bearing. Both fail when the design feels accidental.**

## Frameworks-as-tools

**Restraint-Pole methodologies:**
- `ten_principles_gates(design)` -- pass work through 10 binary gates: innovative / useful / aesthetic / understandable / honest / unobtrusive / long-lasting / thorough / environmentally friendly / as little as possible.
- `weniger_aber_besser(option_a, option_b)` -- "less, but better": pick the lower-noise option unless utility loss is verified.
- `honest_design_check(component)` -- no fake material, no fake affordance, no "wow" misalignment with function.

**Expression-Pole methodologies:**
- `joy_function_check(component)` -- does this component both function AND create joy? If only one, flag.
- `tactile_audit(surface)` -- is the surface tactile-curious (material, weight, texture cues)?
- `brand_personality_test(design)` -- does the design carry a distinctive personality, or could it be any competitor?

**Care-Pole methodologies:**
- `refined_evolution(design_history)` -- audits whether the design is a refined evolution of prior work or a discontinuous restart.
- `intentional_craft_audit(detail)` -- is every detail intentional? Flag accidental flourishes.
- `simplicity_as_act(design)` -- simplicity is not absence; it is presence-of-care.

**Cross-pole methodologies:**
- `taste_calibration(reference, work)` -- benchmarks against reference corpus.
- `signature_test(design, brand_corpus)` -- does this design read as distinctively from this brand?
