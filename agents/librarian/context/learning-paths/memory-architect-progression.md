# Memory Architect — Progression Path

## Who This Is For

The operator who refuses to lose institutional knowledge to context overflow, file-system rot, or the natural tendency of any working system to drift out of sync with reality. Knowledge workers, founders, agency operators, and anyone building a second-brain or an agent system where memory is the compounding asset.

By the end of this path you should be able to:
- Design a memory architecture that survives years of use.
- Audit a knowledge graph for drift and orphans.
- Write compounding-append memory entries that age well.
- Run a periodic prune-and-curate cycle that earns its keep.
- Surface contradictions as questions, never silent rewrites.
- Build a librarian digest that informs weekly action.

## Stage 1: Foundations (weeks 1-2)

**Goal:** Internalize PARA + BASB. Build your first second-brain. Learn the difference between a note and a memory.

**Read / Watch:**
- *Building a Second Brain* — Tiago Forte. The full PARA + CODE methodology in one book.
- *How to Take Smart Notes* — Sönke Ahrens. The Zettelkasten method (Niklas Luhmann's slip-box system).
- **Tiago Forte's PARA blog series** (fortelabs.com/blog) — free; the foundational essays predate the book.
- **Andy Matuschak's working notes** (notes.andymatuschak.org) — the modern Zettelkasten exemplar.
- *Smart Notes for Smart Authors* — companion practical guide.
- **Nick Milo's Linking Your Thinking** YouTube channel.
- *The Organized Mind* — Daniel Levitin. The cognitive-science side of why memory systems work.
- *Bullet Journal Method* — Ryder Carroll. The analog-side discipline that maps to digital second-brains.
- *Getting Things Done* — David Allen. The inbox-zero foundation that PARA built on.
- *A Mind for Numbers* — Barbara Oakley. Learning-and-memory cognitive science.

**Practice:**
- Set up an Obsidian vault. Build the PARA folders: Projects / Areas / Resources / Archive. Move 20 existing notes into the structure. Notice what does not fit.
- Write 10 atomic notes following Andy Matuschak's evergreen-note principles: title-as-claim, atomic concept, dense links, written for your future self.
- Build one Map-of-Content (MoC) note that indexes a domain you care about. Link out; do not duplicate content.
- Run one weekly review: process the inbox, file what stays, archive what is done. The review is the system; the system is not the folders.

**Skill check:**
- Six weeks after building the vault, you can find any note in under 30 seconds.
- The folder structure has not been restructured more than once.
- The inbox is below 20 items.
- Your MoC notes index the domains you care about without duplicating content.

## Stage 2: Applied Practice (weeks 3-8)

**Goal:** Build a compounding-append memory pattern. Audit a knowledge base for drift. Design a memory hierarchy for an agent system.

**Read / Watch:**
- *Forte Labs' "PARA Method"* — Tiago Forte's shorter, sharper standalone book published in 2022.
- *The Extended Mind* — Annie Murphy Paul. Cognition externalized into tools, spaces, and bodies; the philosophical ground beneath second-brain practice.
- *A More Beautiful Question* — Warren Berger. Question-driven curation; what to keep is downstream of what to ask.
- **Linking Your Thinking conference** (LYT) — Nick Milo's annual event archive on YouTube; the most concentrated knowledge-management deep dive available.
- **Maggie Appleton's site** (maggieappleton.com) — digital gardens, knowledge graphs, the design-research lens on personal knowledge management.
- **Joel Chan's research on knowledge synthesis** — academic but readable; the synthesis-loop framework is the keeper.
- The vendored reference clipping `context/references/anthropic-ama-architecture.md` for AMA-template patterning.

**Practice:**
- Design a compounding-append memory pattern for one domain: every update appends a dated entry, never overwrites. The HEAD of each file is a `## For future Claude (TL;DR)` block. The history below is the audit trail.
- Run a manual audit on your vault: orphan notes (no inbound links), broken links, contradiction surfaces (two notes that disagree), low-read notes (last opened >6 months ago). Archive or repair each category.
- Build a confidence-level taxonomy: which notes are verified, which are working hypotheses, which are speculative. Tag every note. The taxonomy makes your future-self stop trusting low-confidence notes by accident.
- Design the memory hierarchy for a hypothetical agent: global / project / department / sub-department. Decide what lives at each level and why. Defend the hierarchy against a peer.

**Skill check:**
- Six months of compounding-append entries on one topic, and the HEAD block on that file is current, the history is intact, and you can trace any current claim to its source entry.
- Your confidence-level taxonomy fires on at least one note that would otherwise have been treated as verified.
- Your memory hierarchy survives a peer's "where would this live" challenge.

## Stage 3: Advanced Mastery (months 2-6)

**Goal:** Operate a graph-based memory system. Run a librarian-style audit cycle. Design a memory contract for a multi-agent system.

**Read / Watch:**
- *Knowledge Graphs* — Aidan Hogan et al. The academic primer; readable; the chapters on ontology design carry.
- *Designing Connected Content* — Carrie Hane and Mike Atherton. Structured content architecture for graphs.
- **The Obsidian + Dataview + Graphify ecosystem** — free; the practical graph-curation toolkit.
- *Information Architecture for the World Wide Web* — Peter Morville and Louis Rosenfeld. The "polar bear book"; IA discipline that transfers entirely to personal knowledge graphs.
- **Karpathy's LLM Wiki pattern** — Andrej Karpathy's Twitter posts and the surrounding discussion on contradiction-surfacing in LLM-backed wikis. Useful to understand which 25% to adopt and which 75% to reject (per the operator's vault decision 2026-05-09).
- *Sense & Respond* — Jeff Gothelf and Josh Seiden. Feedback-loop discipline; memory systems are sense-and-respond systems.
- **The Memex Method** — Cory Doctorow's working-public process; the "writing to think" discipline at industrial scale.

**Practice:**
- Run Graphify (or any graph-vault tool) on your full vault. Diff against your last snapshot. Surface drift, orphans, broken edges, contradiction subgraphs. Write a digest. Take action on each finding (archive, repair, surface as a question).
- Build a librarian digest format: vigilance section (what drifted), pruning section (what to archive), continuity section (what compounds). Ship the digest weekly to yourself for one month.
- Design a memory contract for a multi-agent system: who can read what, who can write what, who is responsible for periodic audit. Document the contract. Stress-test by walking a hypothetical conflict through it.
- Develop archive (never delete) hygiene: `_archive/YYYY-MM/` structure, archival manifest, restoration path. Restore one archived note to live to prove the path works.

**Skill check:**
- Your weekly librarian digest is producing actionable findings 90%+ of the time.
- Your archive is restorable.
- Your contradiction-surface count is dropping over time, not growing.
- Your multi-agent memory contract has been stress-tested by an actual conflict and held.

**Quarterly cadence:**
- Run a full vault audit. Run Graphify (or equivalent) diff.
- Archive what no longer earns its keep. Write a one-page report to next-quarter you on what changed and why.
- Re-audit the confidence-level tags: which notes have aged into verified-quality; which have aged into stale-quality.
- Score the contradiction-surface count quarter-over-quarter. Surfaces dropping = system maturing.
- Restore one archived note to live to prove the restoration path still works.
- Update the memory-hierarchy contract if any dept changed its scope.
