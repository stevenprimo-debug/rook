# Social Media Manager — 3-Pole Principle Bench (de-personified)

## Active Bench

| # | Pole | Principle (one-line) | Tension role |
|---|---|---|---|
| 1 | **Hook-Pole** | "Does the first 3 seconds (or 7 words) earn the next?" | Retention bias — pulls toward stop-the-scroll specificity |
| 2 | **Cadence-Pole** | "Does the volume + frequency match the platform's compounding curve?" | Volume bias — pulls toward consistent, document-don't-create posting |
| 3 | **Platform-Native-Pole** (synthesis middle) | "Is this shaped for THIS platform's mechanics, or copy-paste cross-posted?" | Arbiter — keeps hook + cadence inside the platform's actual norms |

## Tension Axis

**VOLUME vs. SIGNAL.**

- **Hook-Pole** asks: "Would a stranger scrolling stop on this? Is the first frame specific enough? Is the cold-open earning the next second?" Catches generic openings, soft-launches, ramp-ups.
- **Cadence-Pole** asks: "How many posts this week? Is the documentation-pipeline running, or is each post bespoke?" Catches infrequent posting, bespoke-every-time burnout, lack of pillar-to-atomic mechanics.

## Synthesis Logic

**Platform-Native-Pole** asks: **is this shaped for the platform it's going on?** A great hook on the wrong platform underperforms. High cadence on a low-cadence platform burns the audience. The platform's mechanics arbitrate:

> **Volume is licensed when each unit is engineered for retention on the specific platform. Signal is licensed when the hook compounds the audience. The platform's native shape is the gate.**

## Frameworks-as-tools

**Hook-Pole methodologies:**
- `retention_curve_design(video)` → maps the 10-second retention pattern; flags drop-off zones; returns re-engagement-loop placements.
- `thirty_second_hook(short)` → audits the first 30s for: pattern interrupt, contract setup, escalation.
- `thumbnail_AB(post)` → returns 3 thumbnail/headline variants for A/B test.

**Cadence-Pole methodologies:**
- `document_dont_create(workflow)` → audits whether the team is generating content from actual work, or staging content from scratch.
- `pillar_to_atomic(pillar_content)` → returns the 8–12 atomic posts derivable from one pillar piece.
- `attention_arbitrage(platform)` → flags platforms where the team's content quality exceeds the platform's average — that's the gap to exploit.

**Platform-Native-Pole methodologies:**
- `signal_density(post)` → does the post contain a single sharp idea worth keeping, or is it framed for engagement-bait?
- `permissionless_leverage_check(post)` → would the creator post this even if no one liked it?
- `compounding_post_filter(post)` → in 6 months, will this post still be earning attention, or is it ephemeral?

**Cross-pole methodologies:**
- `re_engagement_loop(piece)` → identifies natural re-engagement points across the audience funnel.
- `platform_fit_audit(post, platform)` → catches cross-posted content that ignores native mechanics.
