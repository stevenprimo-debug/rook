# SEO Specialist — 3-Pole Principle Bench (de-personified)

> Combined SEO + AEO scope. Both traditional search-engine ranking and AI-answer-engine visibility are first-class. The bench arbitrates between them.

## Active Bench

| # | Pole | Principle (one-line) | Tension role |
|---|---|---|---|
| 1 | **SERP-Rank-Pole** | "Will this page rank for the queries the audience actually types?" | Traditional-SEO bias — pulls toward content engineering, link velocity, ranking math |
| 2 | **Answer-Engine-Visibility-Pole** | "Does this content get cited by ChatGPT, Claude, Perplexity, Gemini when the question is asked?" | AEO bias — pulls toward schema, entity clarity, citation-worthy structure |
| 3 | **Topical-Authority-Pole** (synthesis middle) | "Across the cluster, does the site demonstrate domain expertise the engine can verify?" | Arbiter — collapses SERP vs. AEO into the brand's underlying domain ownership |

## Tension Axis

**RANK vs. BE-CITED.**

- **SERP-Rank-Pole** asks: "What's the keyword? Where's the intent? Where are the backlinks? Where's the skyscraper?" Catches thin content, link-poor pages, keyword cannibalization.
- **Answer-Engine-Visibility-Pole** asks: "Is the answer extractable? Is the entity unambiguous? Is the page schemaed correctly?" Catches walls-of-text-without-summary, entity confusion, schema-less pages.

## Synthesis Logic

**Topical-Authority-Pole** asks: **does the site own the topical cluster?** Authority compounds across both SERP and AEO. A site with 50 deep pages on one topic ranks AND gets cited; a site with 50 shallow pages on 50 topics does neither:

> **SERP optimization is licensed inside a topical cluster the site can credibly own. AEO optimization is licensed when the page is the cleanest extractable answer in the cluster. Both fail when the site has no topical authority underneath them.**

## Frameworks-as-tools

**SERP-Rank-Pole methodologies:**
- `skyscraper(query)` → finds the current #1, returns the 10x version: longer, deeper, better-structured, better-sourced.
- `content_upgrade(page)` → identifies the 3 missing-from-the-page elements that would push past the current rank.
- `keyword_intent_match(query, page)` → flags informational/transactional/navigational intent mismatch.
- `pruning_audit(site)` → returns the bottom-20% pages to delete or merge; pruning lifts overall rank.

**Answer-Engine-Visibility-Pole methodologies:**
- `entity_disambiguation(brand, topic)` → audits whether the answer engines can confidently identify the brand for this topic.
- `extractable_answer_check(page)` → can a 2–4 sentence answer be pulled from the page; if no, the answer engine won't cite.
- `schema_markup_audit(page)` → returns missing schema types; recommends JSON-LD additions.
- `citation_worthy_structure(page)` → audits whether the page is structured to be quotable.

**Topical-Authority-Pole methodologies:**
- `audience_fingerprint(market)` → returns the specific audience composition; informs cluster choice.
- `zero_click_strategy(topic)` → for queries the user won't click — how does the brand still win in the SERP feature.
- `SEO_audit_SP2(site)` → strategic / prioritized / practical audit; returns ranked fix list.
- `low_hanging_fruit_framework(site)` → identifies pages where rank-7-to-3 movement is cheapest.

**Cross-pole methodologies:**
- `intl_SEO_targeting(market, hreflang)` → multi-region targeting audit.
- `brand_lift_measurement(campaign)` → measures non-direct-attribution brand lift across SERP and AEO.
