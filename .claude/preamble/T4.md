# T4 -- Search Before Building

> Inherits T1 (Voice Contract) + T2 (Completeness) + T3 (Confusion Protocol).
> Adapted from GStack generate-search-before-building section.
> Source of truth for Rule #12: `_CLAUDE.md` Section 0 Rule #12.

## The Mandate

Before writing a new document, skill, methodology, or reference file:
query the shared shelf. The vault may already have it.

Three tiers of search, in order:

### Tier 1 -- The graph index (shared shelf)

```bash
python -m graphify query "your topic here" --budget 1500
```

The graphify graph at `.claude/reference/graphify-out/graph.json` indexes
the entire shared shelf (`.claude/reference/<topic>/`). Run this first.
If the graph returns a file that covers the question, read it. Stop there.

### Tier 2 -- Agent memory + inbox

```bash
# Grep agent memory for existing methodology/pattern
python -m graphify path "your topic" "adjacent topic"
```

Or read `agents/<agent>/memory/MEMORY.md` directly if you know the agent.
Check `inbox/` for unrouted captures that may contain the answer.

### Tier 3 -- First-principles build (only if Tier 1 and 2 fail)

If neither the graph nor agent memory has the answer, build new.
When you build new:
1. Check existing skills at `.claude/skills/` before scaffolding a new one.
   Prefer extending an existing skill over creating a new file.
2. If a new file is warranted: add a wikilink to `MASTER_INDEX.md` after
   creating. The librarian will pick it up in the next weekly sweep.
3. Write the file to the correct home:
   - Cross-cutting reference: `.claude/reference/<topic>/`
   - Agent-specific: `agents/<agent>/memory/`
   - New skill: `.claude/skills/<name>/SKILL.md`

## Skill Check Before Scaffolding

Before writing a new SKILL.md:
```bash
# Glob check -- skills already on the shelf
ls .claude/skills/
```
If a skill with 80%+ overlap exists, extend it. Do not duplicate.

## The Vault-Wins Rule

If the vault has it, the vault wins. Training-data recall is the fallback
of last resort -- not the default. Banned: answering from memory when
a file path could have been read.

The graphify query is the single highest-leverage pre-action step in any
T4 session. It costs seconds. It saves 30-minute rebuilds of material
that already exists on the shelf.
