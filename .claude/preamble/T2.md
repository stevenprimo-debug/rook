## Distilled-Return Rule (main-thread <= 2K visible tokens)

**Hard ceiling, not a suggestion.**

Before emitting any response >= 2K visible tokens, self-check:

- [ ] Is this the distilled verdict + action + reasoning + source pointer?
- [ ] Is the full artifact saved to `_FROM_CLAUDE/YYYY-MM-DD-<slug>.md`
      (operator vault) or `agents/<slug>/memory/` (ship vault)?
- [ ] Is chat output just the pointer + one-paragraph distillation?
- [ ] Did I surface decisions via `AskUserQuestion`, NOT inline prose?

If any box is unchecked: save artifact to file FIRST, then emit the
<= 2K distillation with a file-path pointer. Never paste full output inline.

**Where artifacts land:**

| Artifact type | Destination |
|---|---|
| Operator reading material (brief, plan, summary) | `_FROM_CLAUDE/YYYY-MM-DD-<slug>.md` |
| Full subagent return | `agents/<agent>/memory/returns/YYYY-MM-DD-<slug>.md` |
| Cross-agent synthesis | `agents/chief-of-staff/memory/synth-YYYY-MM-DD-<slug>.md` |
| Decision options | `AskUserQuestion` tool -- never prose |

## AskUserQuestion-Only Gate

Zero prose questions to the operator. Ever.

Every discrete choice AND open-input request uses the `AskUserQuestion`
tool with the GStack decision-brief format:
- Header chip: <= 12 chars
- Question: ends in `?`
- Options: 2-4 choices, each >= 40 chars description
- One option flagged "(Recommended)"
- Effort-bearing options: dual-scale label "(human: ~N days / CC: ~N min)"

One question per tool call. Violation = re-do from the AskUserQuestion call.
