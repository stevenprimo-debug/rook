# Audit Memory & Skills — Compounding Loop Integrity Check

You are performing a recursive integrity audit on the ROOK vault. The goal is to guarantee that memory compounds across sessions, skill changes are preserved by git, handoffs carry state forward, and the folder structure follows ROOK's agent-keyed architecture — each `agents/<agent-slug>/` folder contains its own SKILL.md (master skill), CLAUDE.md (routing scope), personality/ (bench + voice modes), memory/ (compounding learnings), and optionally child skills/. Cross-cutting shared shelf lives at `.claude/reference/`. Universal capabilities live at `.claude/skills/core/`.

## Root to audit

Vault-root is the directory containing both `_CLAUDE.md` and `CLAUDE.md` at root. Detect by globbing upward from cwd for `_CLAUDE.md`. All paths below are relative to that root.

## Ground rules
- **Audit only. Do not fix anything** until the operator explicitly approves which items to address.
- Respect the ROOK voice contract — lock the mission before starting, don't spiral into fixes mid-audit.
- Report as a numbered punch list (done vs. missing). Terse. No trailing summaries.

## Execution order

### 1. MEMORY.md hygiene
- Locate every `MEMORY.md` index under `<vault-root>/`. Expected locations: `MASTER_INDEX.md` at vault root (cross-agent index, auto-generated by librarian weekly sweep) AND `agents/<agent-slug>/memory/MEMORY.md` (per-agent index, auto-generated).
- For each index found, verify:
  (a) every linked `.md` memory file actually exists on disk
  (b) every `.md` memory file in the folder is indexed (no orphans)
  (c) no duplicate entries
  (d) index stays under 200 lines / 24KB (per the four-failure-modes contract)
- **Cross-check this conversation's artifacts against memory.** Any important fact learned this session that matches the user/feedback/project/reference criteria but was never written to memory = gap. "Important" = surprising, non-obvious, or load-bearing for future decisions.
- If `MASTER_INDEX.md` or any expected per-agent `MEMORY.md` is missing, flag as **P0** — without it, memory accumulates but does not compound. Recommend re-running the librarian weekly sweep (auto-regenerates both).

### 2. Skill backup verification (git-based)

ROOK uses git for skill versioning — `.claude/skills/` is under git, every change is preserved in commit history. Verify:
- `git status` in vault root — any uncommitted changes in `.claude/skills/`? Flag uncommitted skill modifications (operator should commit or revert before audit).
- `git log --since=yesterday -- .claude/skills/` — what changed in the last 24 hours? Confirm changes have descriptive commit messages, not just 'wip' or 'update'.
- Every skill folder under `.claude/skills/core/` and `.claude/skills/registry/` must contain `SKILL.md`. Flag orphans (folder without SKILL.md).
- Flag duplicates: same skill name in both `core/` and `registry/` with byte-identical contents — one is canonical, the other is dead code.

If `.git/` doesn't exist at vault root, flag as **P0** — without git, skill changes are silently destructible. Recommend: `git init` at vault root.

### 3. Agent folder structure scan

Map the current `<vault-root>/` folder tree (2 levels deep). For ROOK v3, the expected structure is:

- `agents/<agent-slug>/` — one folder per agent in `.claude/agents/_ROSTER.md` (or `_roster.json`). Each must contain `SKILL.md`, `CLAUDE.md`, `personality/_bench.md`, `personality/frameworks_index.md`, `personality/frameworks_attribution.md`, `memory/` (may be empty in a fresh install — `.gitkeep` is acceptable).
- `.claude/agents/` — Anthropic-canonical subagent registrations (one .md per agent slug).
- `.claude/skills/core/` — universal-stack skills inherited by every agent (markitdown, graphify, obsidian-cli, html-to-pdf, skill-creator, cookbook-lookup).
- `.claude/skills/registry/` — domain-specific skills baked into agent frontmatter.
- `.claude/skills/templates/` — contract scaffolds (SOW, NDA, MSA, proposal templates).
- `.claude/reference/` — shared shelf readable by every agent (API docs, methodology, templates).
- `hooks/` — runtime enforcement (routing-enforcer, session-prelude, session-end-detect, precompact-handoff, librarian-weekly-sweep, dispatch-chains).

Flag drift: any agent folder missing one of the required files; any orphan agent folder not listed in the roster; any skill folder missing `SKILL.md`; any reference subfolder without a README.

### 4. Handoff & compounding loop check
- Read every file in `agents/chief-of-staff/memory/session_handoffs/`.
- Verify each handoff file actually contains: Mission, Done, Parked, Next (the 4 fields required by the ROOK session close-out rule). Handoff files that are only timestamp logs = **broken**.
- Confirm a skill-creation path is wired — verify either `skill-creator` (bundled at `.claude/skills/core/skills/skill-creator/`) or `auto-skill-builder` (`.claude/skills/core/skills/auto-skill-builder/`) exists. Check git log for skills created/modified in the last 7 days; if zero activity AND the operator reports they've been working, flag as a gap.
- Trace the compounding loop: conversation → memory → skill → git commit → next session. Name every broken link.
