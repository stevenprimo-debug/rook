Common KYC patterns extracted

Progressive disclosure with step indicators: Almost every flow shows progress — Neo Financial (3-dot stepper), DoorDash (1 of 4 bar), N26 (linear progress). Never let users feel lost in regulatory hell.

Pre-flight intro screen before camera: Monzo, Qantas, Monese, Kraken, N26 all set expectations before opening the camera — what you'll do, why, lighting requirements, data handling. Reduces drop-off massively.

Trust signals embedded inline: "Powered by Berbix/Onfido/Mitek" encryption notes, privacy policy links — banking apps especially over-index on this (Monzo's lock icon + "encrypted, stored securely", Qantas' explicit "we won't store your passport image").

Document-type selection first, then scan: Pick passport vs ID vs driver's license, then go to camera. Reduces re-scans.

Camera UI conventions: Dark background, rectangular guide frame with corner brackets, real-time edge detection (BlaBlaCar's green border on success), instructional text below the frame, large capture button, manual entry escape hatch always visible.
